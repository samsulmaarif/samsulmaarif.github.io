Perintah Dasar Linux (2)

Prompt shell di Linux ada 2, $ dan #

  • $ : adalah prompt untuk user biasa.

    Contohnya

    samsul@blankon:~$
  • # : adalah prompt untuk pengguna dengan hak akses root.

    Contohnya

    root@blankon:~#

Perintah Dasar Linux (3)

Format perintah Linux standar mempunyai format sebagai berikut:

$ nama_perintah [pilihan] [argumen]

Contoh:

$ ls tanpa argumen
$ ls -a option adalah -a=all, tanpa argumen
$ ls /bin tanpa option, argumen adalah /bin
$ ls /bin /etc /usr ada 3 argumen
$ ls -l /usr 1 option dan 1 argumen
$ ls -la /bin /etc 2 option -l dan -a dan 2 argumen

Perintah sudo dan su

  • sudo
    merupakan perintah yang dapat digunakan untuk menjalan perintah dengan hak akses "root" (super user) oleh pengguna biasa. Seorang administrator dapat mendelegasikan tugas administratifnya kepada pengguna lain tanpa memberikan password root kepada pengguna tersebut.
  • su
    hampir sama dengan sudo, perbedaannya adalah saat menjalankan perintah ini password yang digunakan adalah password root, bukan password pengguna biasa. Jadi administrator harus memberikan password root kepada pengguna yang akan menjalankan perintah ini.

Perintah sudo dan su (2)

Contoh penggunaannya :

  • sudo
    samsul@blankon:~$ sudo poweroff
[sudo] password for samsul:
  • su
    samsul@blankon:~$ su -c poweroff
Password:

Manajemen User dan Group

Kelompok User dan Groups

  • User Admin/Super User
    Yang dimaksud dengan super user adalah user yang bertanggung jawab penuh terhadap sistem dengan tugas yang tidak terbatas. Beberapa tugas super user diataranya adalah mengkonfigurasi sistem, melakukan update sistem, melakukan manajemen user dan group, dan sampai menghapus sistem secara keseluruhan..
  • User Sistem
    User sistem adalah kelompok user yang menjalankan aplikasi daemon user sistem tersebut. User sistem memiliki UID dan GID di atas 0 dan di bawah 500. contoh user sistem beberapa diataranya adalah mail, news, lp, sshd, apache, dan lain sebagainya.

Manajemen User dan Group (2)

Kelompok User dan Groups

  • User/pengguna biasa
    Yang dimaksud dengan user/pengguna biasa adalah user yang dibuat oleh user root dengan UID dan GID diatas 500. User pengguna memiliki wewenang hanya pada home direktori user tersebut sehingga apabila salah satu pengguna mengalami error tidak mempengaruhi user yang lain.

Manajemen User dan Group (3)

Membuat User

  • Untuk membuat user dan group hanya dapat dilakukan oleh super user ‚Äúroot‚Äù. Perintah untuk membuat user baru adalah adduser[spasi]nama_user sedangkan untuk mengisi password user yang dibuat passwd[spasi]nama_user.

    root@blankon:~# adduser haryani

Manajemen User dan Group (4)

Menghapus User

  • Untuk menghapus user jalankan perintah berikut ini: 

    root@blankon:~# userdel haryani
  • atau

    root@blankon:~# userdel -r haryani
Catatan: menghapus dengan menggunakan opsi -r, akan menghapus
secara keseluruhan home direktori user tersebut.

Manajemen User dan Group (5)

Membuat Group

  • Membuat group dengan nama manager 

    root@blankon:~# groupadd manager
  • Menentukan Administrator dan member group manager

    root@blankon:~# gpasswd -A haryani -M samsul,ubuntu manager
Catatan: opsi -A adalah user administrator adalah ‚Äúharyani‚Äù, dan

Manajemen User dan Group (6)

Membuat Group

  • Memberikan password group manager 

    root@blankon:~# gpasswd manager
  • Menambah dan menghapus member dari group manager

    root@blankon:~# gpasswd -a maarif
Adding user maarif to group manager
root@blankon:~# gpasswd -d maarif
Removing user maarif from group manager
Catatan: opsi -a menambah user baru, dan
opsi -d menghapus user.
